# 百合小窝 / Yuri Nest 使用说明

## 启动

在项目目录运行：

```powershell
npm run dev
```

浏览器打开：

```text
http://127.0.0.1:5173/
```

## 当前默认模型

本机通过 `.env.local` 接入 OpenAI-compatible 中转。具体的 `AI_BASE_URL`、`AI_MODEL`、`AI_API_KEY` 配置见 `.env.local.example` 和 `项目文档/PROJECT_HANDOFF.md` 第 4 节。

密钥只放在本机 `.env.local`，不会提交到版本库。如果中转对中文不稳定，可设置 `AI_ESCAPE_UNICODE_CONTENT=true` 兼容层。

也可以在网页"模型 / 数据"页用模型保险箱保存多组模型配置；此时密钥加密保存在服务器 SQLite 里，前端不接触原文。

## 怎么体验

1. 左侧是固定功能区：聊天、记忆、世界树、回收、模型、设置都从这里切换。
2. 右侧是当前功能的主工作区；在“聊天”里直接输入消息即可。
3. 默认按 Enter 发送，按 Ctrl + Enter 换行；也可以在“设置”里切换成 Enter 换行。
4. “记忆 / 世界树 / 模型 / 设置”会在右侧整页打开，页面本身固定，只让内容区域滚动。
5. 手机浏览器左滑或系统返回会先回到应用内部上一页，例如从设置回到聊天；再返回才退出网页。
6. 需要备份时进入“模型”页点“导出”，换浏览器或重装后可再“导入”。

## 小提示

- 说“记住”“以后”“我喜欢”“我不喜欢”“别忘了”时，应用会尝试捕捉一条长期记忆。
- “整理最近聊天”会把当前角色最近几条用户消息手动沉淀进记忆。
- “后台整理”会检查记忆花园，把明显重复的记忆合并，并保留来源和版本记录。
- “花园体检”会自动显示记忆总数、待确认数量、平均可信度、边界记忆、修剪建议和最容易被想起的记忆，方便体验时判断记忆系统有没有变脏。
- “记忆空间”会显示全局、当前角色、项目和世界记忆的数量；如果关系记忆误放到全局，会直接提醒，避免多角色串戏。
- “候选记忆审核”会集中显示 AI 想记但还没确定的内容，可以保存生效、先编辑、暂存归档或删除；确认前不会进入聊天提示。
- “冲突提醒”会提示明显重复、偏好相反或空间放错的记忆；空间放错可以一键移到当前角色关系，重复项可以先归档后一条。
- “最近调用”会显示每次聊天实际注入了哪些长期记忆；如果回复不对味，可以先看这里有没有被旧记忆带偏。
- 聊天页顶部的“本轮记忆”会显示当前角色和当前话题准备调用的长期记忆分组；切换雾岛怜、林秋实时，角色私有记忆会跟着换，不会默认共享主陪伴体的关系记忆。
- 每条记忆现在都能看到类型、状态、记忆空间、敏感等级、可信度、来源原文、版本记录和最近调用时间；误改后可以在“来源与版本”里回滚。
- 每条记忆都有“提及时机”：可自然提起、相关时使用、问起再提、只做边界。这样 AI 不会为了证明自己记得很多而频繁翻旧账。
- 记忆也可以设置“冷却到某个时间”。冷却期间不会进入聊天提示，适合刚纠正过、暂时不想被提起、或者需要观察的内容。
- 点“编辑”可以修改记忆空间：全局用户、当前关系、角色私有、项目、世界、世界分支、当前会话和临时空间都能手动切换。
- 记忆卡片上会出现安全的快捷迁移按钮，例如“当前角色关系”“当前角色私有”“项目空间”“设为全局”“相关时提”“问起再提”“冷却7天”，方便测试时快速纠错。
- 点每条记忆的“档案”可以打开完整记忆档案，集中查看内容、标签、来源证据、版本线和调用记录。
- 自动捕捉到的关系类记忆会先进入“待确认”，确认前不会进入聊天提示；禁忌/边界类记忆会优先保护当前对话。
- “设置 / 记忆系统”可以关闭自动捕捉，或调高自动记忆门槛，避免无关闲聊越积越多。
- 记忆和世界树都可以编辑、删除；删除后先进入“回收花园”，可以恢复、单条彻底删除，或一键清空。彻底删除会留下不含原文的遗忘标记，避免旧内容被自动捕捉复活。
- 回收花园默认保留 30 天；也可以在“设置”里改成永久保存，或自定义 1-365 天。
- 当前是本地优先版本，浏览器本地会保存数据；需要跨设备时，到“模型”页使用云端读取/保存。
