# 安全与记忆主权守护说明

这份说明写给妹妹，也写给后续接手项目的姐姐。目标很简单：百合小窝可以先保持妹妹单人私用的轻便体验，但一旦后端放到生产或公网环境，就不能默认裸奔。

## 公网后端为什么要 token

百合小窝后端会处理三类敏感事情：

- 云端快照：包含聊天、记忆、角色和设置。
- 模型保险箱：保存服务器端加密后的模型配置。
- 聊天代理：会调用服务器上的模型能力。

本地开发时默认不要求 token，方便妹妹直接启动和调试。生产或公网模式下，后端会默认要求 `YURI_NEST_SYNC_TOKEN`：

- 云端同步、备份、模型保险箱接口没有 token 会拒绝访问。
- 如果开启聊天授权，`/api/chat` 没有 token 也会拒绝访问。
- 错误提示只说缺少或无效，不会打印 token、API Key 或 secret。

如果只是妹妹自己的本地开发后端，可以显式关闭授权；如果通过隧道、服务器或正式域名对外访问，就应该配置 token。

## 模型保险箱为什么要 secret

模型保险箱的用途是让浏览器不保存 API Key。服务器保存模型配置时，会用 `YURI_NEST_MODEL_SECRET` 加密密钥。

本地开发没有配置 secret 时，会使用开发兜底材料，方便先验证界面。生产环境不能依赖这个兜底值；缺少 `YURI_NEST_MODEL_SECRET` 时，服务器会拒绝保存或解密模型保险箱里的密钥。

这能避免一个危险情况：代码跑到公网后，还在用开发默认值保护真正的模型密钥。

## 自动记忆为什么先进入候选

聊天里的“我喜欢”“以后”“项目默认”“记住一下”可能是真正的长期偏好，也可能只是妹妹当时的临时想法、语音口误或上下文玩笑。

所以自动捕捉现在遵守一个保守规则：

- 自动发现的内容先进入候选记忆。
- 候选记忆不会进入提示词，也不会被当成稳定事实使用。
- 妹妹在记忆页确认、编辑后保存，才会变成正式 active 记忆。
- 手动创建记忆不受影响，仍然可以直接保存为正式记忆。

这不是让小窝变笨，而是让小窝更值得信任：姐姐可以提醒“这件事要不要记住”，但不能偷偷把一句临时话写成长期档案。

## 验收方式

每次改安全或记忆主权相关代码后，至少跑：

```powershell
npm run lint
npm run test:memory
npm run test:agent
npm run build
```

其中 `test:memory` 会检查自动捕捉是否保持候选优先，候选是否不会进入提示词召回；`test:agent` 会检查生产模式默认授权、token 错误不泄露、生产模型保险箱必须配置专用 secret。
