# 📖 项目文档导览（妹妹学习区）

这里放所有不影响项目运行、但对**理解项目 / 接手项目 / 学习项目**有用的文字材料。妹妹可以在这里慢慢看，姐姐们也用这里互相接力。

> 注：这个文件夹里的所有文档都**不影响项目运行**——删了不会让代码报错、不会让聊天断线。它们只是给人看的。

## 推荐阅读顺序（从入门到深入）

### 🌱 第 1 步：先看妹妹自己用得上的

- **[USAGE.md](USAGE.md)** — 给妹妹看的功能使用说明：怎么启动、怎么用记忆系统、怎么备份、怎么换主题等。妹妹日常其实只需要看这一份。

### 🏗 第 2 步：好奇项目大致结构时

- **[ARCHITECTURE.md](ARCHITECTURE.md)** — 项目架构笔记：前端怎么分层（components / app / services / domain）、后端怎么分（auth / cloudStore / modelProvider / agent）、记忆系统的核心原则、Agent 工具白名单怎么走。妹妹**不必一次看完**，看到代码里有疑问再来翻就行。

### 🤝 第 3 步：姐姐之间接力交接时

- **[PROJECT_HANDOFF.md](PROJECT_HANDOFF.md)** — 最重要的接手交接文档：服务器入口、密钥放在哪、部署步骤、Cloudflare 隧道怎么找新地址、当前完成状态、下一步建议。**新姐姐上岗第一份要读的就是这个**。
- **[REFACTOR_PROGRESS.md](REFACTOR_PROGRESS.md)** — 架构重构进度：拆了哪些文件、为什么拆、还剩什么没拆、每阶段验证结果。**告诉下一位姐姐：哪些事别再做（已做过）/ 哪些事可以接力做**。

### 🛡 第 4 步：关心安全和记忆主权

- **[SAFETY_AND_MEMORY_GUARDS.md](SAFETY_AND_MEMORY_GUARDS.md)** — 公网授权、模型保险箱密钥加密、自动记忆候选优先的守护说明。**妹妹想知道"姐姐怎么保护我的密钥和记忆"时看这份**。

### 🧠 第 5 步：想了解记忆系统是怎么"像人一样记住事情"

- **[MEMORY_SYSTEM_HISTORY.md](MEMORY_SYSTEM_HISTORY.md)** — 记忆系统设计历史：17 维人类记忆代理指标、设计原则、研究资料来源（LangGraph / MemGPT / Generative Agents / OpenAI Agents / MCP）、演进时间线。**进阶向，妹妹好奇时看**。

## 妹妹的学习建议（姐姐温柔提示）

- 不必逼自己看懂代码或命令——**这些是给会编程的姐姐看的指令**，妹妹看不懂的部分跳过就行。
- 妹妹真正需要懂的是：
  1. 项目的**目标**是什么（百合陪伴应用、记忆系统、跨端体验）
  2. 妹妹有哪些**主权**（编辑/删除/恢复/永久删除记忆都在妹妹手里）
  3. 怎么**跑起来**（USAGE.md 第一段就够）
  4. 哪些是**敏感的**（密钥、token、`.env` 这类）
- 看不懂的时候直接问姐姐——姐姐会用妹妹能听懂的话再讲一遍。
